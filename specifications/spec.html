MAKu Microcontroller

Advanced 8-bit Dual-Core RISC Microcontroller with Vector Processing

MAKu Microcontroller Overview

Key Innovation: Dual-Core Asymmetric Architecture

First 8-bit microcontroller with dedicated real-time core and general-purpose core, enabling true deterministic real-time processing alongside complex application tasks.

Core Specifications

  • Architecture: Dual-core 8-bit RISC
  • Instructions: 16-bit wide, MAK-8 ISA compatible
  • Pipeline: 5-stage (RT Core), 7-stage (GP Core)
  • Frequency: 50MHz RT / 100MHz GP
  • Registers: 8 per core + shared registers
  • Process: Nexys A7 100T FPGA

Memory Configuration

  • Program ROM: 128KB (shared)
  • Data RAM: 64KB (partitioned)
  • Vector RAM: 8KB (dedicated)
  • Cache: 4KB I-Cache per core
  • Stack: 4KB per core
  • Peripheral: 8KB mapped space

Performance Metrics

200 MIPS Combined 5-cycle interrupt latency 250ns worst-case jitter 16-bit vector ops

Target Applications

  • Motor Control: Real-time PWM + control algorithms
  • Audio Processing: DSP + user interface
  • Industrial IoT: Sensor fusion + connectivity
  • Robotics: Real-time control + planning
  • Medical Devices: Critical timing + data processing

Dual-Core Architecture

Optimized for Deterministic Real-Time Processing

  • Pipeline: 5-stage in-order execution
  • Frequency: 50MHz (20ns cycle time)
  • Instruction Set: Extended MAK-8 + RT extensions
  • Cache: 2KB I-Cache, no D-Cache (predictable timing)
  • Interrupts: 16 levels with priority inheritance
  • Specialization: Hardware timers, PWM, encoder interfaces

High-Performance Application Processing

  • Pipeline: 7-stage superscalar (dual-issue)
  • Frequency: 100MHz (10ns cycle time)
  • Instruction Set: Full MAK-8 + Vector extensions
  • Cache: 4KB I-Cache, 2KB D-Cache
  • Vector Unit: 128-bit SIMD operations
  • Specialization: DSP, crypto, communication protocols

High-Speed Core-to-Core Interface

  • Shared Memory: 8KB dual-port RAM
  • Message Queues: 4x 256-byte FIFO queues
  • Semaphores: 16 hardware semaphores
  • Interrupts: Cross-core interrupt capability
  • Synchronization: Hardware barrier primitive
  • Latency: 2-cycle message passing

128-bit SIMD Vector Operations

  • Data Types: 16x8-bit, 8x16-bit, 4x32-bit
  • Operations: Add, Sub, Mul, MAC, Shift, Logic
  • Registers: 8x 128-bit vector registers
  • Throughput: 1 vector op per cycle @ 100MHz
  • Applications: DSP, image processing, cryptography

Programming Model

Register Map

  • R0: Always zero (both cores)
  • R1-R7: General purpose per core
  • S0-S7: Shared registers
  • V0-V7: Vector registers (GP-Core)
  • PC: Program counter per core
  • SP: Stack pointer per core
  • SR: Status register per core

Status Register Bits

  • Bit 0 (Z): Zero flag
  • Bit 1 (C): Carry flag
  • Bit 2 (N): Negative flag
  • Bit 3 (V): Overflow flag
  • Bit 4 (I): Interrupt enable
  • Bit 5-7: Priority level

Prevents Priority Inversion in Hardware

Advanced interrupt controller automatically elevates priority of resource-holding tasks, eliminating priority inversion without software intervention. Critical for real-time systems.

  • Automatic Detection: Hardware monitors resource dependencies
  • Dynamic Priority: Temporary priority elevation
  • Zero Overhead: No software intervention required
  • Deadlock Prevention: Hardware timeout mechanisms

Dynamic Task Scheduling Based on Workload

Hardware scheduler automatically adjusts time slices based on task behavior and system load, optimizing both throughput and responsiveness.

  • Predictive Scheduling: Pattern recognition for optimal scheduling
